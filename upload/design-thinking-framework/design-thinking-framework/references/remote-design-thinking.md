## Remote Facilitation Adjustments

### Before the Session

**1 Week Before:**
- Send calendar invite with clear objectives
- Share pre-work (research review, persona reading)
- Test technology with all participants
- Create and test whiteboard templates

**1 Day Before:**
- Send reminder with direct links
- Confirm roles (facilitator, notetaker, timekeeper)
- Prepare backup plan for tech failures

**30 Minutes Before:**
- Open all tools and test
- Set up breakout rooms
- Queue up templates
- Start recording (with permission)

### Facilitation Techniques

**Energy Management:**
```
REMOTE ENGAGEMENT TACTICS

Every 15-20 minutes, do one of:
- Quick poll ("Thumbs up if you agree")
- Chat waterfall ("Type your answer, hit enter on my count")
- Reaction check ("Show me your energy with an emoji")
- Stretch break ("Stand up for 30 seconds")
- Volunteer spotlight ("[Name], share your screen and show us...")

Camera Fatigue Mitigation:
- Cameras optional during solo work
- Cameras on during share-outs
- Regular breaks (5 min per hour minimum)
```

**Participation Balance:**
```
ENSURING ALL VOICES

SOLO → PAIR → GROUP Pattern:
1. Solo work (silent, everyone on mute, sticky notes)
2. Pairs discuss (breakout rooms, 3-5 min)
3. Group share (main room, rapid share-outs)

Prevents:
- Loudest voice dominating
- Groupthink
- Zoom fatigue from constant talking

Round-Robin for Share-outs:
"Let's hear from everyone. [Name], you're first. Keep it to 30 seconds.
When you finish, call on the next person."
```

**Technical Facilitation:**
```
MANAGING THE HYBRID ATTENTION

- Name what you're doing: "I'm now sharing the whiteboard..."
- Use annotation tools live
- Point with cursor, not words: "This sticky note here" while pointing
- Check in frequently: "Can everyone see this? Give me a thumbs up."
- Have a co-facilitator monitor chat

When Things Break:
- Have phone numbers for critical participants
- Prepare a "tech failure" slide with backup instructions
- Know how to quickly switch to backup tool
```

## Remote Activity Adaptations

### Empathy Interviews (Remote)

```
REMOTE INTERVIEW SETUP

TECH CHECK (5 min before):
- Confirm video/audio working
- Ask permission to record
- Confirm quiet environment

SCREEN SHARE ALTERNATIVES:
- If showing prototype: Share screen
- If discussing experiences: Keep cameras on faces
- If drawing journey: Use whiteboard collaboration

BODY LANGUAGE READING:
Remote makes body language harder. Compensate by:
- Asking directly: "How did that make you feel?"
- Watching facial micro-expressions closely
- Noting hesitations and tone changes
- Using silence prompts (count to 5 after they answer)

RECORDING BEST PRACTICES:
- Record video and audio
- Use auto-transcription (Otter.ai, Rev)
- Take timestamped notes for key moments
- Capture screenshots of expressions
```

### Empathy Mapping (Remote)

```
REMOTE EMPATHY MAP SESSION

SETUP:
- Pre-build empathy map template in Miro/FigJam
- Assign sticky note colors by participant
- Create duplicate templates for group work

FACILITATION:
1. Share research summary (5 min, presenter shares screen)
2. Solo sticky noting (5 min, everyone on mute)
   - Each person adds to their color
3. Affinity clustering (5 min, one person screen-shares, others direct)
4. Synthesis discussion (10 min, cameras on)

REMOTE VARIATION:
Instead of quadrants, use a simpler structure:
- "What did we hear?" (SAY)
- "What did we see?" (DO)  
- "What might they think/feel?" (THINK/FEEL)
- "What are their PAINS and GAINS?"
```

### Crazy 8s (Remote)

```
REMOTE CRAZY 8s

OPTION 1: Digital Native
- Each person has a Crazy 8s frame in shared whiteboard
- Use shape/drawing tools
- Timer on screen
- No talking during generation

OPTION 2: Paper + Photo
- Everyone has paper folded into 8 squares
- Draw on paper (more natural)
- Timer on screen
- When done, take photo and upload to board

OPTION 3: Hybrid
- Sketch on paper
- Take photo
- Upload
- Present from digital board

SHARE-OUT:
- 1 minute per person
- Facilitator controls timer on screen
- "Spotlight" feature to focus attention
```

### Dot Voting (Remote)

```
REMOTE DOT VOTING

MIRO/FIGJAM:
- Use voting plugin or emoji stamps
- Lock frames so ideas don't move
- Set vote limit (3 votes typical)
- Results auto-tally

SIMPLE ALTERNATIVE:
- Number each idea (Idea 1, Idea 2, etc.)
- Everyone types votes in chat
- "Type your 3 votes in chat. Don't hit enter until I say go."
- Count votes from chat log

ANTI-BIAS TECHNIQUE:
- Keep voting anonymous
- No discussion before voting
- Reveal all votes simultaneously
```

### Prototype Testing (Remote)

```
REMOTE USER TESTING PROTOCOL

SETUP:
- Send prototype link in advance (or share screen live)
- Ask participant to share their screen
- Confirm recording permission
- Test audio/video quality

MODERATION ADJUSTMENTS:
- Can't see full body language—ask more feeling questions
- Harder to stay silent—use mute strategically
- Technology may distract—allocate extra time

SCREEN SHARING OPTIONS:
1. Participant shares (best for realistic flow)
2. Facilitator shares with remote control
3. Facilitator shares and takes direction

CAPTURE METHODS:
- Record session (Zoom, Lookback)
- Take timestamped notes
- Screenshot key moments
- Use observer note-taking template in shared doc

POST-TEST DEBRIEF:
- Immediately after, 5-min team huddle
- "What stood out?"
- "What surprised us?"
- "What should we change for next test?"
```

## Remote Workshop Agendas

### 2-Hour Remote Sprint

```
REMOTE DESIGN SPRINT (2 hours)

PRE-WORK (async, 30 min):
- Watch research summary video
- Review persona document
- Post initial HMW ideas to board

HOUR 1:
00-10: Welcome, ground rules, icebreaker
10-20: Research share-out (facilitator presents)
20-35: Empathy map creation (solo + group)
35-50: Journey map walk-through
50-60: POV statement crafting (breakout pairs)

BREAK (10 min)

HOUR 2:
00-10: HMW brainstorm (solo sticky notes)
10-15: HMW voting
15-25: Crazy 8s (digital or paper)
25-35: Share top ideas (1 min each)
35-45: Dot voting + selection
45-55: Next steps planning
55-60: Checkout + feedback
```

### Full-Day Remote Workshop

```
FULL-DAY REMOTE WORKSHOP

PRE-WORK (async, day before):
- Complete pre-reading
- Add research observations to board
- Test all technology

MORNING SESSION (3 hours):

09:00-09:15: Opening, agenda, ground rules
09:15-09:30: Icebreaker activity
09:30-10:00: Research download + Q&A
10:00-10:45: Empathy mapping (breakouts)
10:45-11:00: BREAK
11:00-11:30: Journey mapping
11:30-12:00: POV + HMW development

LUNCH BREAK (1 hour)

AFTERNOON SESSION (3 hours):

13:00-13:15: Energizer + recap
13:15-13:35: Crazy 8s ideation
13:35-14:00: Idea sharing + voting
14:00-14:15: BREAK
14:15-15:00: Concept development (breakouts)
15:00-15:30: Concept presentations
15:30-15:50: Prototype planning
15:50-16:00: Wrap-up + next steps
```

## Async Design Thinking

When teams can't meet synchronously, use async methods.

### Async Empathy Research

```
ASYNC RESEARCH PROTOCOL

DAY 1-3: DIARY STUDY
- Send participants daily prompts via text/email
- "Today, note every time you encounter [problem]"
- "How did you feel? What did you do?"
- Collect responses in shared form

DAY 4-5: ARTIFACT COLLECTION
- "Send photos of your [workspace/tools/etc.]"
- "Screenshot your [app/process/workaround]"
- Compile in shared folder

DAY 6: VIDEO RESPONSE
- Send 3-5 questions via Loom
- Participants record 2-min video responses
- Team watches and notes insights
```

### Async Ideation

```
ASYNC IDEATION SPRINT

PHASE 1: SOLO GENERATION (48 hours)
- Share HMW statement
- Everyone generates 10+ ideas individually
- Post to shared board without seeing others

PHASE 2: REVEAL + BUILD (24 hours)
- Unlock all ideas at same time
- Add comments: "Building on this: what if..."
- Combine and evolve ideas

PHASE 3: VOTING (24 hours)
- Each person gets 5 votes
- Vote for ideas (including others')
- Anonymous until deadline

PHASE 4: SELECTION (sync call, 30 min)
- Review top-voted ideas
- Quick discussion
- Select for prototyping
```

## Hybrid Workshop Tips

When some participants are in-room and some are remote.

### Setup Requirements

```
HYBRID WORKSHOP CHECKLIST

TECHNOLOGY:
☐ Room camera showing full whiteboard
☐ Individual laptops for digital whiteboard access
☐ High-quality room microphone
☐ Screen sharing the digital whiteboard on room TV
☐ Remote participants can see AND hear room clearly

EQUITY PRINCIPLES:
- All sticky notes go on digital board (not physical wall)
- In-room participants use laptops, not just physical tools
- Facilitator references remote participants by name
- Designate "remote advocate" in room
- Remote participants present as often as in-room

ANTI-PATTERNS TO AVOID:
✗ Physical whiteboard that remote can't see
✗ Side conversations in room
✗ Forgetting to check on remote participants
✗ In-room people dominating discussion
```

### Facilitation Balance

```
KEEPING HYBRID EQUITABLE

START REMOTE:
"Let's hear from our remote participants first on this one."

CHECK-INS:
Every 15 minutes: "Remote team, what questions do you have?"

PARTICIPATION TRACKING:
Keep tally of who has spoken
Actively invite quieter voices

NAME BEFORE SPEAKING:
"This is [name] in the room..." helps remote know who's talking

REMOTE ADVOCATE ROLE:
One in-room person's job is to:
- Watch chat continuously
- Speak up for remote participants
- Ensure camera shows relevant activity
- Flag audio/video issues immediately
```

## Remote Design Thinking Challenges & Solutions

| Challenge | Solution |
|-----------|----------|
| Zoom fatigue | More breaks, cameras optional during solo work, shorter sessions |
| Low energy | Frequent energizers, variety of activities, music during work time |
| Unequal participation | Solo before group, breakouts, round-robin, chat waterfalls |
| Technology failures | Backup tools ready, phone numbers collected, practice runs |
| Hard to read room | More verbal check-ins, emoji reactions, direct questions about feelings |
| Whiteboard chaos | Locked templates, assigned areas, clear naming conventions |
| Time zone issues | Async activities, recorded sessions, rotating meeting times |
| Building on ideas | "Yes, and..." in chat, dedicated build-on rounds, combination time |
| Prototyping remotely | Figma collaboration, paper-to-photo, video prototypes |
| Testing remotely | Participant screen share, recorded sessions, detailed observation notes |
